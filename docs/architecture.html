🧳 TravelAgent — 아키텍처

Evidence 기반 멀티에이전트 해외여행 플래닝 MVP. FastAPI 백엔드 + React 프론트엔드. 핵심 규칙: agent → tool → connector → provider, provider 출력은 Evidence로 정규화한 뒤에만 계획에 반영.

핵심 설계 규칙
  • 계층 규칙: agent → tool → connector → provider, Evidence 정규화 후에만 계획 반영
  • 중앙 상태: 단일 TripPlanState를 패치, JSON 스냅샷으로 버전 관리(추적·재개)
  • Mock-first: ENABLE_LIVE_PROVIDERS=false(기본), LLM도 기본 off
  • 안전 게이트: 예약은 approval 후 시뮬레이션만, 가격상한·payload 해시·PII 동의 강제
